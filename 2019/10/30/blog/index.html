使用hexo搭建自己的博客并部署到github

Author: Crazy-Y Date: October 30, 2019  12:16:48

安装前提

Node / Git

Node.js (Node.js 版本需不低于 8.6，建议使用 Node.js 10.0 及以上版本)
Git

安装hexo

(所有必备的应用程序安装完成后，即可使用 npm 安装 hexo)
全局安装hexo

运行

项目目录

_config.yml

网站的主要配置文件，可以配置你网站的大部分信息

package.json

应用数据。

scaffolds

模板目录，你新建一篇博客的时候会以这个目录中的模板文件来创建。

source

资源目录，你新建的博客都存储在这里。在生成页面的时候，hexo会忽略掉命名为 _ 开头的文件或文件夹， _post 除外，会将 .md 和 .html 后缀的文件按主题生成页面，并复制到public目录下，其他后缀的文件则直接复制过去，不进行额外转换操作。

themes

主题目录，你可以为自己的网站挑选好看的主题，然后在 _config.yml 文件中指定主题即可。

修改配置

_config.yml

更多配置请查询hexo官网

配置github仓库

前提

拥有github账号
如果你还没有github的账号，那需要先注册一个

建立github仓库

在github上创建一个仓库，命名为 username.github.io username就是你的github用户名，我仓库名字是 JieYangAdmin.github.io，并且需要勾选Initialize this repository with a README；
https://xxxxxx.github.io/ 就是你的博客的默认地址 (如：https://jieyangadmin.github.io/)

连接hexo和git

设置你的用户名称与邮件地址

在你的文件目录下右键点击 Git Bash Here

使用ssh-keygen生成私钥和公钥

如果之前有私钥和公钥，就不必在此生成

登录Github，点击头像下的settings，添加ssh,新建一个new ssh key，将id_rsa.pub文件里的内容复制上去。
输入ssh -T git@github.com，测试添加ssh是否成功。如果看到Hi后面是你的用户名，就说明成功了

修改配置_config.yml

部署到github

在你的文件目录下右键点击 Git Bash Here

部署到git我们需要安装hexo-deployer-git插件

上传

搞定收工~
进入 https://xxxxxx.github.io/ 就是你的博客 (如：https://jieyangadmin.github.io/)
