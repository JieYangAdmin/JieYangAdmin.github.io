MongodDB

  • MongoDB 是为了快速开发互联网Web应用而设计的数据库系统
  • MongoDB 的设计目标是极简、灵活、作为web应用栈的一部门
  • MongoDB 的数据模型是面向文档的，所谓文档是一种类似于JSON的结构
    (MongoDB这个数据库中存的是各种各样的JSON)

概念

  • 数据库(database)
    数据库是一个仓库，可以用来存放集合
  • 集合(collection)
    集合类似于数组，在集合中可以存放文档
  • 文档(document)
    文档数据库中最小单位，我们存储和操作的内容都是文档

下载安装

MongoDB的版本偶数版本是稳定版，奇数版本是开发版

  • 配置全局环境变量
    path -> C:\Program Files\MongoDB\Server\4.0\bin

  • 创建文件夹

    1. 在 C:/ 创建 “ data “ 文件夹

    2. 在 C:/data/ 创建 “ db “ 文件夹（存放数据库内容）

    3. data文件夹自选位置: mongod –dbpath XXX(路径: D:\data\db)

    4. mongod –dbpath XXX(路径) –port 端口号(修改端口号:最大不要超过65535)

运行

mongod 运行MongoDB服务器(最小化终端，不要关闭)
mongo 连接MongoDB(新开一个终端)

数据库

  1. 数据库的服务器
    服务器用来保存数据
  2. 数据库的客户端
    客户端用来操作服务器，对数据进行增删改查的操作

基本命令

show dbs : 显示当前所有数据库
use XXX : 进入XXX数据库，如果没有，自动创建并进入
db : 显示当前所处的数据库
show collections : 查看当前数据库的所有集合

数据库的CRUD(增删改查)的操作

  1. 插入
    db.XXX.insert(xxx) : 向 XXX 集合中 插入 xxx 文档
    ex: 向 test 数据库中的 stus 集合中插入一个对象{ name: “crazy-Y” , age: 25, sex: “男” }

  1. 查询
    db.XXX.find({}) : 查询当前集合当中的所有文档
    db.stus.find({“XXX”:”xxx”}) : 根据对象条件查询
    db.stus.find({}).count() : 查询stus集合内文档的数量
    db.stus.find({“XXX” : { $gt : xxx }}) : 查询stus集合中XXX的值大于 xxx 的所有文档
    db.stus.find({“XXX” : { $gte : xxx }}) : 查询stus集合中XXX的值大于等于 xxx 的所有文档
    db.stus.find({“XXX” : { $lt : xxx }}) : 查询stus集合中XXX的值小于 xxx 的所有文档
    db.stus.find({“XXX” : { $lte : xxx }}) : 查询stus集合中XXX的值小于等于 xxx 的所有文档
    db.stus.find({“XXX” : { $gt : xxx1 , $lt : xxx2}}) : 查询stus集合中XXX的值大于 xxx1 小于 xxx2 的所有文档
    db.stus.find({}).skip(xx).limit(XX); : 查询指定区间的文档, skip(): 跳过的条数 limit():显示的条数
    ex: db.stus.find({}).skip((页数 - 1) * 每页显示条数).limit(每页显示条数) 分页功能

  2. 修改
    update() : 默认只会修改查询到的第一个

  • db.XXX.update(查询条件,新对象) : update 默认 新对象 替换 旧对象
    ex: db.stus.update({“name”:”crazy-J”},{“sex”:”男”})

  • db.XXX.update(查询条件,{$set:{XXX: xxx}}) : 将查询到的文档的XXX属性值改为xxx
    $set : 修改文档的指定属性,如果没有该属性则为添加
    $unset : 删除文档的指定属性

ex: db.stus.update({“name”:”crazy-J”},{$set:{
“sex” : “男”
}})

ex: db.stus.update({“name”:”crazy-J”},{$unset:{
“sex” : “男”
}})

  • db.XXX.update(查询条件,{$set:{XXX: xxx}},{multi : true}) : 修改所有的符合查询条件的文档
    ex: db.stus.update({“sex”:”男”},{$set:{
    “year” : “2020”
    }},{“multi” : true})
  1. 删除
    db.XXX.remove({}) : 删除集合内所有的文档
    db.XXX.remove({“XXX”:”xxx”}) : 删除符合条件的所有文档
    db.XXX.remove({“XXX”:”xxx”}，true) : 删除所有符合条件的第一个文档

  • 清空集合
    db.XXX.remove({}) : 性能差
    db.XXX.drop()

  • 删除数据库
    db.dropDatabase()

sort 和 投影

  1. sort()
    指定文档的排序规则，需要传递一个参数来指定排序规则。（1 是升序, -1 是降序）
    ex: db.XXX.find({}).sort({“age”: 1 }) 按照年龄从小到大排序
    ex: db.XXX.find({}).sort({“age”: -1}) 按照年龄从大到小排序
    ex: db.XXX.find({}).sort({“age”: 1 , “num”: -1}) 先按照年龄从小到大排序，如果年龄一样，就按照num从大到小排序

  2. 投影
    在查询时，可以在第二个参数的设置位置来设置查询结果的投影
    ex: db.XXX.find({}, {“username”: 1, “age”: 1}) 只查看符合条件的所有文档的username( 1代表 显示 ，0 代表 不显示 )

mongoose

好处

  • 可以为文档创建一个模式结构(Schema)
  • 可以对模型中的对象/文档进行验证
  • 数据可以通过类型转换为对象模型
  • 可以使用中间件来应用业务逻辑挂钩
  • 比Node原生的MongoDB驱动更容易

新对象

  • Schema(模式对象)
    Schema对象定义约束了数据库中的文档结构

  • Model
    Model对象作为集合中的所有文档的表示，相当于MongoDB数据库中的集合collection

  • Document
    Document表示集合当中过得具体文档，相当于集合中的一个具体的文档
